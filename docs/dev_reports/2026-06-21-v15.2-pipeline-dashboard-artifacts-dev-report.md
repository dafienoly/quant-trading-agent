## 功能说明

V15.2 为 Agent Pipeline 增加可下载、可离线查看的验收与诊断产物：严格回归结果写入 `.agent/reports/pipeline_report.json`，再渲染为中文静态页面 `.agent/reports/pipeline_dashboard.html`。运行期目录被 Git 忽略，只通过 GitHub Actions artifact 交付。

新增 `pull_request` 触发的 `Agent PR 轻量验证` workflow。它在 PR opened、synchronize、reopened、ready_for_review 和 edited 时运行，覆盖严格 pipeline 回归、目标 pytest、`git diff --check`、交易敏感路径检查和 `.agent/tmp`/`.agent/reports` 已跟踪文件检查，从而避免 PR 没有 checks 记录。

Stage runner、main merge gate 和 PR validation 都在 `always()` 路径生成并上传诊断 artifact。正式回归未产出 JSON 或 viewer 失败时，workflow 生成明确标记为 `diagnostic_fallback` 的失败报告，不会冒充正式 PASS。

PR validation 维护一条带稳定 marker 的中文 Bot 评论，说明如何从最新 Actions 运行查看 artifact。该评论同时声明 artifact 仅用于诊断与人工审阅，不代表允许合并。
