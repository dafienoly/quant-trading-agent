# V15.3 DeepSeek Agent Runtime 重实现功能说明报告

## 1. 结论

已基于最新 `main` 重新实现统一 DeepSeek Agent Runtime，并完成 BugFixAgent 迁移。PR #3 的模块拆分、fake client 和 tool loop 思路被作为参考，但没有直接合并旧分支；V15.3 验证发现的 schema、缓存、工具和失败关闭缺口均在本实现中重新处理。

## 3. 实现范围

### 3.1 Runtime 核心

- `src/llm/deepseek_runtime.py`
  - async-first 调用和同步兼容入口。
  - timeout、瞬时错误有限重试、非瞬时错误立即失败、取消传播和并发限制。
  - Thinking Mode、JSON Output、只读工具循环和 usage 提取。
  - 未知 profile/schema/tool、空内容、非法 JSON、schema mismatch 全部 fail closed。
- `src/llm/schemas.py`
  - 统一请求、结果、profile 和业务输出 Pydantic 模型。
  - BugFix analysis 强制 `needs_human_review=true`。
  - BugFix proposal 强制 `requires_approval=true`。
- `src/llm/model_router.py`
  - 保留现有调用接口，实际请求统一委托 Runtime。

### 3.2 会话、缓存、工具与观测

- `src/llm/conversation.py`
  - UUID 会话、多轮消息、递归脱敏、7 天 TTL。
  - 默认不持久化 `reasoning_content`。
- `src/llm/context_cache.py`
  - 缓存稳定 system/profile/schema 前缀并生成 fingerprint。
  - 实际调用使用缓存前缀；不保存用户 Bug prompt。
  - 并发首次写入使用锁保护。
- `src/llm/tool_registry.py`
  - 只读工具白名单、参数 schema、路径逃逸阻断、输出截断与脱敏。
  - 固定 ripgrep 命令，不开放任意 Shell、Git、写文件、网络或交易工具。
- `src/llm/usage.py`
  - 线程安全记录延迟、tokens、cache hit/miss、工具轮次和调用状态。

### 3.3 BugFixAgent 迁移

- 删除业务层 OpenAI client 和 `_call_deepseek()`。
- analyze/propose_fix 使用 Runtime 和严格 schema。
- 删除非法 JSON 回退 `raw_analysis` 的旧私有路径。
- 保留人工审批、受限模块阻断、工作区保护、测试失败回滚和 Git 失败关闭逻辑。

### 3.4 配置

`.env.example` 新增：

- `LLM_TIMEOUT_SECONDS`
- `LLM_MAX_RETRIES`
- `LLM_MAX_CONCURRENCY`
- `LLM_TOOL_MAX_ROUNDS`
- `LLM_THINKING_DEFAULT`
- `LLM_THINKING_EFFORT`
- `LLM_CONTEXT_CACHE_DIR`
- `LLM_CONVERSATION_DIR`

## 7. 安全确认

- `chat.completions.create` 只存在于 `src/llm/deepseek_runtime.py`。
- BugFixAgent 不再直接导入或调用 OpenAI client。
- LLM Runtime 不导入券商、执行、订单、账户或风控实现。
- 未修改交易敏感模块，未新增真实交易或自动下单能力。
- BugFix 提案继续要求人工审批，受限模块继续二次阻断。
- 未提交密钥、`.agent/tmp/**` 或 `.agent/reports/**`。
- 未自动合并 `main`。

## 8. 剩余风险

- 当前验证环境为 Python 3.14；项目声明支持 Python 3.10+，CI 仍需覆盖仓库标准 Python 版本。
- DeepSeek API 参数兼容性只通过 fake client 契约验证，真实服务连通性需要单独使用受控密钥验收，不能替代本次离线安全测试。
