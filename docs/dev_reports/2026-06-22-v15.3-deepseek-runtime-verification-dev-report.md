### 4.2 当前主线实现

PR #3 中声明的以下文件未进入 `main`：

### 4.3 BugFixAgent 仍为直连

`src/product_app/bug_fix_agent.py` 仍然：

- 直接导入 `OpenAI`。
- 在 `_call_deepseek()` 内直接创建 OpenAI 兼容 client。
- 直接调用 `client.chat.completions.create()`。
- 只通过 `ModelRouter().get_config()` 读取配置，没有委托统一 Runtime。

### 4.4 失败路径

- 缺少密钥：`ModelRouter.chat_json()` 返回 `missing_api_key`，该子项满足拒绝继续调用。
- 超时：ModelRouter 和 BugFixAgent 的 client 调用均无显式 timeout 配置，未满足可控超时和 fail-closed 契约。
- 空内容：BugFixAgent 将空字符串解析为 `{"raw_analysis": ""}`，没有返回明确失败状态，未满足 fail closed。

### 4.5 测试覆盖

当前只有 `tests/test_model_router.py` 覆盖基础配置、缺包和状态接口。主线不存在 PR #3 中的六个 Runtime 专项测试文件，因此无法证明 schema、工具、缓存、会话、用量和 BugFixAgent 迁移能力。

## 6. 实现建议

1. 关闭或标记 PR #3 为已被后续重实现取代，保留历史审计价值。
2. 从最新 `main` 创建新的 Runtime 实现分支。
3. 重新定义统一异步 Runtime、schema 校验、只读工具、缓存、会话和用量接口。
4. 将 BugFixAgent 的所有模型调用迁移到 Runtime，禁止直接 client 调用。
5. 为 missing key、timeout、empty content、无效 schema、工具越权和取消请求补充 fail-closed 测试。
6. 保持 LLM 不可直接下单，不引入任何真实交易能力。

## 7. 安全确认

- 未修改任何交易敏感模块。
- 未新增真实交易或真实下单能力。
- 未允许 LLM 直接决定买卖或调用下单路径。
- 未使用真实模型密钥或发起真实模型请求。
- 未提交 `.agent/tmp/**` 或 `.agent/reports/**`。
- 未自动合并 `main`。

## 8. 剩余风险

- PR #3 的思路可能仍有可复用部分，但与当前主线相差 47 个主线提交，直接合并具有较高回归和契约漂移风险。
- 当前开发环境的项目依赖仍不完整，完整现有测试需在安装完整依赖的 CI 或标准开发环境中复跑。
- 在重实现完成前，任何文档、UI 或接口都不应宣称当前产品已具备统一 DeepSeek Agent Runtime。
