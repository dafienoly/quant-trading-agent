# V15.2 Pipeline Dashboard Artifacts 架构设计

## 架构摘要

V15.2 复用现有 `scripts/agent_pipeline_regression.py` 作为 pipeline report JSON 的事实来源，复用 `scripts/agent_pipeline_report_viewer.py` 将该 JSON 渲染为静态 HTML。GitHub Actions 只负责编排、失败兜底、artifact 上传和 PR 评论，不引入新的在线服务或交易侧依赖。

## 需求映射

| 需求 | 模块 | 实现 |
|---|---|---|
| F-001/F-002 | `scripts/agent_pipeline_regression.py`、`scripts/agent_pipeline_report_viewer.py` | 严格回归写 JSON，viewer 渲染 HTML；workflow 提供诊断 fallback |
| F-003/F-004 | `.github/workflows/*.yml` | `always()` 生成并上传 artifact，维护带 marker 的中文 PR 评论 |
| F-005/F-006/F-007 | `.github/workflows/agent-pr-validation.yml` | PR 事件触发五类轻量验证 |
| F-008 | `scripts/agent_pipeline_report_viewer.py` | 用户可见 dashboard 文案中文化 |
| F-009 | merge gate 测试 | 明确禁止 `gh pr merge` |

## 模块边界

### 回归报告生成

`scripts/agent_pipeline_regression.py` 继续负责确定性检查与退出码。workflow 调用时使用 `--strict --json --output .agent/reports/pipeline_report.json`，即使严格检查失败，脚本仍应先写出 JSON 再返回非零。

### Dashboard 渲染

`scripts/agent_pipeline_report_viewer.py` 只读取本地 JSON、`.agent/state.json`、`.agent/gates/*.json` 和 `docs/` 清单，输出单文件 HTML。它不访问 GitHub、不执行外部命令、不修改仓库状态。

### Workflow 编排

1. `agent-pr-validation.yml` 使用 GitHub-hosted Linux runner，运行轻量、无网络业务依赖的验证。
2. `agent-stage-runner.yml` 与 `agent-main-merge-gate.yml` 在所有前置步骤结束后以 `if: always()` 生成诊断产物。
3. Artifact 上传步骤同样使用 `if: always()`。
4. PR 评论通过稳定 HTML marker 查找并更新已有 bot 评论，避免每次同步提交新增一条评论。

## 数据契约

### `.agent/reports/pipeline_report.json`

```json
{
  "status": "pass|warn|fail",
  "summary": {
    "critical_count": 0,
    "warning_count": 0,
    "info_count": 0
  },
  "checks": [],
  "artifacts": {
    "report_path": ".agent/reports/pipeline_report.json"
  }
}
```

### `.agent/reports/pipeline_dashboard.html`

UTF-8 静态 HTML，页面只展示报告内容，不包含脚本远程加载、密钥、Cookie 或账户信息。

## 失败处理

1. 回归脚本失败但已写 JSON：保留真实失败 JSON，继续渲染 HTML。
2. 回归脚本异常且未写 JSON：workflow 写入明确标记为诊断 fallback 的失败 JSON，不得将其标记为 PASS。
3. HTML viewer 失败：workflow 写入简短诊断 HTML，说明 viewer 失败，不伪装为正式 dashboard。
4. Artifact 上传失败：workflow 保持失败，不静默跳过。
5. PR 评论权限不足：不改变验证结论，但 Actions 日志必须保留错误；artifact 仍可从运行页查看。

## 安全影响

本变更只触及 CI、pipeline 工具、测试和文档。不会修改或调用交易、风控、订单、账户、券商、行情或真实执行模块；不会新增 LLM 下单路径；不会改变 Risk Agent、股票池过滤、人工确认和 fail-closed 行为。

## 测试策略

1. 在 `tests/test_agent_pipeline_automation.py` 对 workflow 源文件做确定性断言。
2. 在 `tests/test_agent_pipeline_regression.py` 验证新回归检查存在并通过。
3. 在 `tests/test_agent_pipeline_report_viewer.py` 验证中文 dashboard 关键文案与输出。
4. 运行用户指定的 strict regression、目标 pytest、`git diff --check`、敏感路径检查和 tracked-file 检查。

## 分支计划

当前实现分支为 `feat/v15.2/pipeline-dashboard-artifacts`，从最新 `main` 创建。完成后只提交 V15.2 相关脚本、workflow、测试和中文报告，由用户人工审阅并决定合并。

## 开发指引

1. 先补 workflow 源断言测试，再新增/修改 workflow。
2. `.agent/reports/` 只在运行期创建，并加入 `.gitignore`。
3. Artifact 生成步骤必须放在会提交 `.agent` 的步骤之后，避免被 `git add .agent` 收录。
4. 保持 `agent-main-merge-gate.yml` 不含自动合并命令。
5. 所有新增用户可见文案使用中文。
