# 回测政策 (BACKTEST_POLICY.md)

> 本文档是 Backtest Agent 的硬约束。任何策略、因子或参数组合在进入实盘盯盘、模拟盘或人工确认交易前，必须按本文档完成验证。
> 回测结论只能说明历史样本表现，不得宣传为稳定盈利能力。

---

## 1. 核心原则

1. **禁止未来函数**：信号生成只能使用信号时点已经可获得的数据。
2. **信号日与成交日分离**：日频信号若使用当日收盘价、收盘因子或盘后数据生成，默认只能在下一交易日成交。
3. **先成本后收益**：所有收益指标必须扣除佣金、印花税、滑点和可配置交易成本。
4. **先可交易性后成交**：涨停不可买、跌停不可卖、停牌不可交易，成交失败必须记录原因。
5. **先样本外后结论**：不得只展示全样本或最佳区间结果。
6. **风控一票否决**：回测中触发风控拒绝时，必须阻止对应交易，并在日志或交易拒绝记录中体现。

---

## 2. 数据要求

回测输入必须来自清洗后的标准行情数据，至少包含：

- `symbol`
- `trade_date`
- `open`
- `high`
- `low`
- `close`
- `volume`
- `amount`
- `pct_change`
- `turnover_rate`
- `is_suspended`
- `is_st`

推荐同时包含：

- `pre_close`
- `limit_up`
- `limit_down`
- `adj_factor`
- `market`
- `sector`

禁止事项：

1. 不得静默填充 `open/high/low/close/volume/amount`。
2. 不得用未来交易日的数据补齐历史信号。
3. 不得在财报公告日前使用财务数据。
4. 不得只使用当前仍存续股票构造历史股票池。

---

## 3. 成交时序政策

### 3.1 日频信号

若信号依赖当日任一收盘后才确定的数据，包括 `close`、当日成交额、当日涨跌幅、当日总评分或盘后更新的基本面/主题数据，则成交时序必须为：

```
T 日收盘后生成信号
T+1 日开盘价或 T+1 日 VWAP 模拟成交
T+1 日涨跌停/停牌状态决定是否可成交
```

### 3.2 成交价模式

| 模式 | 含义 | 是否允许作为默认 |
|------|------|------------------|
| `next_open` | 下一交易日开盘价成交 | 是 |
| `vwap` | 下一交易日 VWAP 或近似 VWAP 成交 | 是 |
| `close` | 信号日收盘价成交 | 否，仅允许对照测试 |
| `open` | 信号日开盘价成交 | 否，仅允许测试低频开盘前信号 |

`next_open` 和 `vwap` 不得使用信号日行情作为成交行情。

### 3.3 盘中信号

Phase 4 以后若引入分钟线或实时行情，必须记录：

- 信号触发时间
- 使用的最后一根 K 线时间
- 订单可成交时间
- 行情延迟
- 数据源

盘中信号不得用尚未收盘的未来分钟线计算指标。

---

## 4. 成本与市场规则

每次成交必须计算：

1. 双边佣金
2. A 股卖出侧印花税
3. 港股交易税费差异
4. 滑点
5. 最低佣金

成交可行性必须检查：

1. 停牌
2. 涨停买入失败
3. 跌停卖出失败
4. A 股整手数量
5. 可用现金
6. 可卖持仓
7. 股票池范围

---

## 5. 必输出指标

回测报告必须输出 AGENTS.md 3.5 要求的全部指标：

- `annual_return`
- `max_drawdown`
- `sharpe_ratio`
- `calmar_ratio`
- `win_rate`
- `profit_loss_ratio`
- `turnover`
- `cost_adjusted_return`
- `benchmark_return`
- `excess_return`
- `monthly_return`
- `yearly_return`

同时必须输出：

- 交易次数
- 总佣金
- 总印花税
- 总滑点
- 总交易成本
- 交易明细
- 每日净值
- 拒绝成交记录或拒绝原因

---

## 6. 样本外与稳健性要求

策略进入下一阶段前，至少完成：

1. 样本内测试
2. 样本外测试
3. 分年度测试
4. 牛市、熊市、震荡市测试
5. 交易成本敏感性测试
6. 参数敏感性测试
7. 统计显著性检验

禁止：

1. 只展示最佳年份或最佳参数。
2. 用全样本调参后宣称策略有效。
3. 样本外明显劣化仍进入实盘盯盘。
4. 忽略换手率、回撤、成本后的收益。

---

## 7. Phase 准入门槛

### 7.1 进入 Phase 4 前

必须满足：

1. 回测引擎无信号日/成交日前视偏差。
2. `next_open` 和 `vwap` 使用下一交易日行情。
3. 全量测试通过。
4. 已知未修复问题必须记录风险等级和处理阶段。
5. 回测结果不得被标记为实盘收益。

### 7.2 进入 Phase 5 前

除 Phase 4 验收外，还必须满足：

1. 模拟盘或纸面交易日志可追踪。
2. 实时数据延迟报告可用。
3. Risk Agent 返回 `risk_pass=false` 时订单草稿无法进入确认队列。
4. 人工确认页面不提供一键全量确认。

### 7.3 进入 Phase 6 前

必须同时满足 ROADMAP_AND_CONSTRAINTS.md Phase 6 的全部启用条件，不得由 Agent 自行放宽。

---

## 8. 审计要求

每次回测代码改动必须补充测试，至少覆盖：

1. 成交时序
2. 成本计算
3. 涨跌停/停牌
4. 风控拒绝
5. 样本外拆分
6. 报告字段完整性

任何修复回测可信度的问题，必须新增回归测试，确保相同偏差不会再次出现。
