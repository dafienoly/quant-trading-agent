# 因子研究指南 (FACTOR_RESEARCH_GUIDE.md)

> 本指南规范量化因子的挖掘、计算、评估与入库流程，确保所有因子具备可解释性和稳健性。
> 所有 Factor Research Agent 必须严格按照本文档执行。

---

## 1. 因子定义标准

每个因子必须包含以下元数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| factor_name | str | 唯一标识，如 `momentum_20d` |
| display_name | str | 中文名称，如"20日动量" |
| category | str | 分类：`technical`, `fundamental`, `sentiment`, `theme` |
| description | str | 因子逻辑的简明描述 |
| formula | str | 数学公式或计算伪代码 |
| data_source | str | 所需数据源及字段 |
| frequency | str | 计算频率：`daily`, `weekly`, `monthly` |
| author | str | 创建者/修改者 |
| created_date | str | 创建日期 |
| version | int | 版本号 |

---

## 2. 因子计算规范

### 2.1 数据预处理

- 所有输入数据必须来自 `data_storage/cleaned`。
- 计算前必须检查：停牌、涨跌停、ST、缺失值。
- 缺失值处理策略必须明确声明（例如：填充 0、前向填充、或剔除），且不得引入未来信息。
- 极值处理：默认使用 MAD（绝对中位差）或百分位截尾（1% / 99%），方法必须记录。

### 2.2 禁止事项

1. **使用未来函数**：因子值在 `t` 时刻计算时，不得使用任何 `t+1` 及之后的数据。
2. **信息泄漏**：财报因子必须使用公告日而非报告期截止日对齐。
3. **幸存者偏差**：回测计算因子表现时，股票池必须包含已退市股票，剔除时点与退市日一致。
4. **前视偏差**：因子预处理（标准化、中性化）只能使用滚动窗口历史数据，不可用全样本。

## 3. 因子评估体系

Factor Research Agent 必须输出以下评估指标，不可仅凭收益高低判断因子有效性。

### 3.1 必输出指标

```yaml
factor_name: "示例因子"
ic_mean: 0.035          # IC 均值
ic_std: 0.12            # IC 标准差
ic_ir: 0.29             # IR = ic_mean / ic_std
rank_ic_mean: 0.04      # Rank IC 均值
coverage: 0.95          # 因子覆盖率（有效值比例）
turnover: 0.18          # 月度单边换手率（基于 top/bottom 分组）
decay_analysis:         # 因子衰减分析（IC 随持有期变化）
  1d: 0.035
  5d: 0.028
  10d: 0.020
  20d: 0.012
long_short_return: 0.15 # 多空年化收益（扣除成本）
long_short_sharpe: 1.2
max_drawdown: -0.08
```

### 3.2 分层回测要求

- 按因子值将股票分为 5 组或 10 组，报告每组年化收益、夏普、最大回撤。
- 多空组合（top - bottom）收益必须扣除双边 0.3% 交易成本。
- 回测时长不少于 3 年，且需覆盖至少一个牛熊周期。

### 3.3 中性化验证

- 必须进行 **行业中性化** 和 **市值中性化** 后的测试，展示中性化前后的 IC 变化。
- 若中性化后 IC 显著下降（>30%），说明因子很大程度由行业或市值驱动，需特别标注风险。

---

## 4. 因子入库标准

一个因子要被策略层使用，必须同时满足：

1. Rank IC 均值 > 0.02（或 < -0.02，负向因子取绝对值）。
2. IC_IR > 0.2。
3. 样本外（最近 1 年）表现未明显劣化（IC 均值与全样本差距不超过 30%）。
4. 多空收益在扣除成本后显著为正。
5. 最大回撤 < 15%。
6. 换手率合理（< 50% 月均单边）。
7. 经行业和市值中性化后仍然有效。
8. 因子文档齐全，包含上述所有元数据与评估报告。

---

## 5. 因子生命周期管理

- 每个因子有状态：`DRAFT` -> `TESTED` -> `APPROVED` -> `ACTIVE` -> `DEPRECATED` -> `ARCHIVED`
- 入库因子进入 `ACTIVE` 状态后，每月需滚动更新一次评估报告。
- 若连续 2 个月 IC 均值低于阈值，因子状态降级为 `DEPRECATED`，策略层自动停止使用。
- 已弃用因子保留在因子库中，但不可参与信号生成。

---

## 6. 当前因子方向指导

前期重点围绕 AI 算力半导体轮动策略，因子应侧重于：

- **技术趋势**：均线排列、新高强度、波动率收缩
- **情绪资金**：成交量比率、相对强度、板块联动
- **政策主题**：半静态主题分，由人工 + LLM 辅助维护
- **基本面**：营收/利润增速、利润率变化、一致预期上调

因子开发优先级：**技术/情绪 > 主题 > 基本面**（因基本面更新频率低）。

---

## 7. 文档与可复现性

- 每个因子必须提供独立可运行的 Python 脚本或 Notebook，包含从数据加载到评估图表的完整流程。
- 随机种子必须固定，以确保结果可复现。
- 因子计算核心逻辑需单元测试覆盖，测试用例应包括：边界值、缺失值、极端行情情景。

---

## 8. 禁止事项汇总

1. 禁止只根据单次回测结果推荐因子。
2. 禁止使用未来数据（财务数据、价格、公告等）。
3. 禁止忽略交易成本（手续费、滑点、印花税、冲击成本）。
4. 禁止忽略停牌、涨跌停和流动性约束。
5. 禁止在全样本上调参后报告结果，必须划分样本内外。
6. 禁止仅展示多空收益而隐藏回撤、换手率等风险指标。
7. 禁止将 LLM 主观判断直接作为因子值，LLM 仅用于辅助解释。
