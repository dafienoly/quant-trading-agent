# 执行政策 (EXECUTION_POLICY.md)

> 本文档规范订单从生成到成交的完整生命周期。Execution Agent 必须严格遵循本政策。
> 默认系统 **不自动下单**，所有实盘交易必须经过显式人工确认。

---

## 1. 核心原则

1. **未确认不下单**：任何 LEVEL_2 及以上的交易指令，必须获得用户逐笔确认。
2. **风控不通过不下单**：Risk Agent 是唯一放行关口。
3. **非交易时间不下单**：仅允许在交易所连续竞价时段生成委托。
4. **不超可用资金/持仓**：买入金额不得超过账户可用资金（扣除手续费），卖出数量不得超过可卖持仓。
5. **可追溯**：每一笔订单必须有完整的日志链路，从信号 ID -> 风控 ID -> 订单 ID -> 成交记录。

---

## 2. 交易模式定义

| 模式 | 常量 | 说明 |
|------|------|------|
| 研究模式 | `LEVEL_0` | 仅数据访问和报告生成，禁用交易相关模块 |
| 信号模式 | `LEVEL_1_SIGNAL_ONLY` | 生成买卖建议，不生成订单，不下单 |
| 人工确认模式 | `LEVEL_2_HUMAN_CONFIRM` | 生成委托单，用户确认后通过券商接口下单 |
| 自动交易模式 | `LEVEL_3_AUTO` | 严格条件下自动下单，必须配置 Kill Switch |

**默认配置：**

```env
ENABLE_LIVE_TRADING=false
REQUIRE_HUMAN_CONFIRMATION=true
MAX_TRADING_LEVEL=LEVEL_1_SIGNAL_ONLY
```

- `ENABLE_LIVE_TRADING=false` 时，所有订单只会发往模拟交易/纸交易，**不会连接真实券商接口**。
- 即使 `ENABLE_LIVE_TRADING=true`，只要 `REQUIRE_HUMAN_CONFIRMATION=true`，每笔订单仍需人工点击确认。

---

## 3. 订单生命周期

```
信号触发
  |
  v
生成订单草稿 (status=CREATED)
  |
  v
风控检查 --(失败)--> 拒绝 (REJECTED)，通知用户
  |(通过)
  v
订单检查（价格、数量、时间、资金）
  |(通过)
  v
人工确认队列 --(超时/拒绝)--> 取消 (CANCELLED)
  |(确认)
  v
发送至券商/模拟引擎 (status=SENT)
  |
  v
等待成交回报
  +-- 全部成交 (FILLED)
  +-- 部分成交 (PARTIALLY_FILLED) -> 剩余部分可撤单或等待
  +-- 拒绝/错误 (REJECTED)
```

---

## 4. 订单生成规则

### 4.1 买入订单

- `price_type` 默认 `LIMIT`。
- `limit_price` 建议为信号触发价 + 合理偏移（不超过 2%）。
- `quantity` 计算必须满足：

```python
quantity = floor(可用资金 * 仓位比例 / (limit_price * (1 + 手续费率)))
quantity = min(quantity, 单票最大仓位限制换算数量)
quantity = 100 的整数倍（A股）
```

- 若计算出的数量 < 100 股，则该订单不生成。

### 4.2 卖出订单

- `quantity` 不得超过当前可卖持仓。
- 止损卖出可使用 `MARKET` 价（若券商支持），但需标记为风控强平。
- 分批卖出：每次卖出不超过持仓的 50%，除非触发清仓信号。

---

## 5. 人工确认协议

- 在 `LEVEL_2_HUMAN_CONFIRM` 下，所有订单进入**待确认队列**。
- 确认界面必须清晰展示：
  - 股票名称、代码
  - 买卖方向
  - 价格、数量
  - 预计金额
  - 触发策略
  - 止损/止盈位
  - 风险提示
  - 风控检查结果
- 提供「确认」「拒绝」「修改价格/数量」三个动作。
- 确认有效期：当日有效，收盘后未确认自动取消。
- **禁止提供「全部确认」「一键确认」功能**，必须逐笔操作。

---

## 6. 真实交易接口适配

- 所有券商接口必须继承 `BrokerAdapter` 抽象类：

```python
class BrokerAdapter:
    def submit_order(self, order: Order) -> OrderResult
    def cancel_order(self, order_id: str) -> CancelResult
    def query_order(self, order_id: str) -> OrderStatus
    def query_positions(self) -> List[Position]
    def query_account(self) -> AccountInfo
```

- 初期实现 `PaperBroker` 模拟交易接口，记录所有操作但不真实成交。
- 后续接入的券商 SDK 必须经完整测试（包括异常、断网、重复回报等场景）。

---

## 7. 交易时段控制

- 允许下单时段：
  - A股：09:30-11:30, 13:00-15:00
  - 港股：09:30-12:00, 13:00-16:00
- 尾盘 5 分钟（A股 14:55-15:00）禁止开新仓买入，只允许卖出。
- 集合竞价时段（09:15-09:25）仅允许模拟，真实交易不接受此期间的订单。
- 系统必须每 30 秒校对一次交易所时间。

---

## 8. 异常处理

- 订单发送后，若 3 秒内未收到回报，进入「待确认」状态，立即查询订单状态。
- 连续 3 笔订单出现网络错误，自动暂停所有交易 15 分钟，并通知用户。
- 成交回报与订单数量不符，触发告警并暂停该股票后续交易。

---

## 9. 模拟交易规范

- 模拟交易必须模拟真实的流动性约束：涨停不买、跌停不卖、停牌不成交。
- 模拟成交价使用 Bar 的 VWAP 或下一根 K 线开盘价（可配置）。
- 模拟交易必须同步计算滑点和手续费。
- 模拟交易日志与实盘日志分开存储，标记 `env=paper`。

---

## 10. 禁止事项

1. 禁止未经 Risk Agent 检查直接发送订单。
2. 禁止在 LEVEL_1 模式下创建订单对象。
3. 禁止跳过人工确认界面（LEVEL_2）。
4. 禁止在非交易时段调用真实下单接口。
5. 禁止对创业板、科创板、ST 股生成任何买入订单。
6. 禁止使用用户未绑定的券商账户发送订单。
