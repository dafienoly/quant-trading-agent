# ROADMAP_AND_CONSTRAINTS.md

## 1. Roadmap 总览

项目分为 6 个阶段：

- **Phase 0**: 项目骨架与约束文档
- **Phase 1**: 数据层与股票池
- **Phase 2**: 因子与策略评分
- **Phase 3**: 回测与评估
- **Phase 4**: 实盘盯盘与信号生成
- **Phase 5**: 人工确认交易
- **Phase 5.5**: 产品化交付 Demo（一键启动、集成前端、自动反馈）
- **Phase 6**: 小资金自动交易实验

默认只推进到 Phase 5.5。Phase 6 必须经过长期模拟验证后再启用。

## Phase 2: 因子与策略评分

### 目标

实现 AI 算力半导体轮动策略评分体系。

### 任务

1. 实现政策主题分
2. 实现情绪资金分
3. 实现基本面分
4. 实现技术趋势分
5. 实现总评分
6. 实现买入信号
7. 实现卖出信号
8. 实现板块轮动评分

### 核心评分

```
total_score = (
    0.25 * policy_score
  + 0.30 * sentiment_score
  + 0.20 * fundamental_score
  + 0.25 * trend_score
)
```

### 买入信号

- `BUY_BREAKOUT`: 趋势突破买入
- `BUY_PULLBACK`: 回踩低吸买入
- `BUY_AMBUSH`: 材料设备埋伏买入

### 卖出信号

- `SELL_HALF_STOP_LOSS`
- `SELL_STOP_LOSS`
- `SELL_TREND_BREAK`
- `SELL_SENTIMENT_FADE`
- `SELL_TAKE_PROFIT_HALF`
- `SELL_TAKE_PROFIT`

### 验收标准

1. 每只股票每天能生成 4 类因子分
2. 每只股票每天能生成总分
3. 能输出买入、卖出、持有信号
4. 每个信号必须有解释文本
5. 信号不允许包含未来数据

## Phase 3: 回测与评估

### 目标

验证策略是否具备历史有效性。

### 任务

1. 实现日频回测框架
2. 加入交易成本
3. 加入滑点
4. 加入印花税
5. 加入涨跌停无法成交逻辑
6. 加入停牌无法成交逻辑
7. 加入仓位管理
8. 加入回测报告
9. 加入样本内、样本外测试
10. 加入不同市场环境测试

### 回测基本假设

- 买入价格：次日开盘价或 VWAP 模拟
- 卖出价格：次日开盘价或 VWAP 模拟
- 手续费：可配置
- 印花税：卖出侧计算
- 滑点：可配置
- 涨停无法买入
- 跌停无法卖出
- 停牌无法交易

### 架构复核补充（2026-06-08）

- Phase 3 起，所有回测必须遵守 `BACKTEST_POLICY.md`。
- 日频信号若使用当日收盘价、当日因子或盘后数据生成，默认只能在下一交易日成交。
- `next_open` 和 `vwap` 不得使用信号日行情作为成交行情。
- `close` 和信号日 `open` 仅允许作为对照测试，不得作为默认回测结论。

### 关键指标

- 年化收益
- 最大回撤
- 夏普比率
- Calmar
- 胜率
- 盈亏比
- 换手率
- 交易次数
- 交易成本
- 超额收益
- 月度收益
- 年度收益

### 验收标准

1. 回测结果可复现
2. 回测包含交易成本
3. 回测包含滑点
4. 回测包含涨跌停限制
5. 回测输出完整报告
6. 样本外结果不严重劣化
7. 最大回撤在可接受范围内

## Phase 4: 实盘盯盘与信号生成

### 目标

接入实时行情，生成盘中提醒，但不自动下单。

### 任务

0. 建立 Phase 4 实时风控骨架：数据延迟、数据空值、候选股范围、交易模式、Kill Switch 状态只读监控
1. 接入实时行情
2. 每 1 到 5 分钟更新行情
3. 监控持仓
4. 监控候选股
5. 监控板块强弱
6. 触发买卖信号
7. 推送提醒
8. 生成盘中日志

### Phase 4 开发顺序要求（2026-06-08 架构裁决）

1. 先实现 `risk_engine` 的运行时风控结果模型与只读 Kill Switch 状态。
2. 再实现实时行情 Provider、行情延迟报告和数据健康门禁。
3. 再实现信号生成服务，所有信号仍为 `LEVEL_1_SIGNAL_ONLY`，不得生成订单对象。
4. 最后实现 API/UI/通知层。

### 盯盘频率

- 普通模式：5 分钟一次
- 活跃模式：1 分钟一次
- 弱网模式：15 分钟一次

### 盘中买入时间规则

- 弱势行情：上午不买，只看 14:30 后尾盘确认
- 强势行情：9:45 后允许第一笔试仓
- 大跌行情：不开新仓
- 连续大涨行情：不追高，只等回踩

### 验收标准

1. 可以盘中获取实时行情
2. 可以监控持仓盈亏
3. 可以监控候选股触发信号
4. 可以生成买卖提醒
5. 所有提醒都有解释
6. 不会自动真实下单

## Phase 5: 人工确认交易

### 目标

Agent 生成订单建议，用户确认后执行。

### 任务

1. 设计订单草稿
2. 接入模拟交易
3. 接入券商交易接口
4. 增加人工确认页面
5. 增加订单二次确认
6. 记录下单日志
7. 记录成交结果
8. 生成交易复盘

### 下单流程

```
信号触发
    ↓
生成订单草稿
    ↓
风控检查
    ↓
订单检查
    ↓
用户确认
    ↓
发送交易指令
    ↓
记录成交
    ↓
生成复盘
```

### 订单必须包含

- 股票代码
- 股票名称
- 买卖方向
- 建议价格
- 建议数量
- 建议仓位
- 触发策略
- 止损位
- 止盈位
- 风险说明
- 用户确认状态

### 验收标准

1. 未确认不能下单
2. 风控不通过不能下单
3. 订单超仓不能下单
4. 非交易时间不能下单
5. 所有订单有日志
6. 所有成交可追踪

## Phase 6: 小资金自动交易实验

### 启用条件

必须同时满足：

1. 模拟盘连续运行不少于 60 个交易日
2. 实盘小资金人工确认交易不少于 30 个交易日
3. 策略最大回撤低于预设阈值
4. 信号误触发率可接受
5. 数据源稳定
6. 交易接口稳定
7. 风控模块无绕过路径
8. 用户手动开启自动交易模式

### 自动交易限制

- 单笔最大金额：账户资产 3%
- 单日最大买入次数：2 次
- 单日最大卖出次数：5 次
- 单日最大亏损：2%
- 账户最大回撤：8%
- 自动交易时间：仅限 9:45 - 14:45
- 尾盘最后 10 分钟不自动开新仓

### Kill Switch

出现以下任一情况，自动交易立即停止：

1. 行情数据延迟超过 60 秒
2. 账户资产异常
3. 委托失败连续 3 次
4. 单日亏损超过 2%
5. 个股异常波动
6. 数据源返回空值
7. 风控模块异常
8. 用户手动停止
9. 系统时间异常
10. 网络异常

### 2. 数据约束

**禁止：**

1. 使用未来数据
2. 使用未复权价格计算历史收益
3. 静默填充关键行情
4. 忽略停牌
5. 忽略涨跌停
6. 忽略数据延迟
7. 使用来源不明的数据
8. 数据源切换后不做一致性校验

**必须：**

1. 保留原始数据
2. 保留清洗后数据
3. 记录数据来源
4. 记录更新时间
5. 输出数据质量报告
6. 支持多数据源替换

---

### 3. 回测约束

**禁止：**

1. 不计交易成本
2. 不计滑点
3. 不计印花税
4. 不计涨跌停无法成交
5. 不计停牌无法成交
6. 只展示最好区间
7. 只展示最好参数
8. 用全样本调参后宣称有效
9. 使用未来财务数据
10. 忽略幸存者偏差

**必须：**

1. 样本内测试
2. 样本外测试
3. 分年度测试
4. 牛市、熊市、震荡市测试
5. 参数敏感性测试
6. 交易成本敏感性测试

---

### 4. 风控约束

默认风控：

- 单只股票最大仓位：15%
- 单个细分方向最大仓位：25%
- 半导体总仓位最大：60%
- 现金最低比例：20%
- 单只股票亏损 5%：减半提醒
- 单只股票亏损 8%：强止损提醒
- 单日账户亏损 2%：停止开新仓
- 单日账户亏损 3%：只允许减仓
- 账户回撤 8%：进入防守模式
- 账户回撤 12%：停止实盘交易

### 5. Agent 约束

Agent 不允许：

1. 自行开启自动交易
2. 自行修改风控阈值
3. 自行删除交易日志
4. 自行删除回测失败结果
5. 自行扩大交易范围
6. 自行加入创业板、科创板
7. 自行使用用户未授权数据源
8. 自行把模拟收益当作真实收益
9. 自行提交密钥
10. 自行绕过人工确认

### 6. LLM 使用约束

**LLM 可以做：**

1. 新闻摘要
2. 政策归因
3. 公告解读
4. 研报摘要
5. 策略解释
6. 信号解释
7. 复盘报告
8. 异常排查

**LLM 不可以直接做：**

1. 直接决定下单
2. 直接修改仓位
3. 绕过量化信号
4. 绕过风控
5. 编造财务数据
6. 编造政策利好
7. 编造新闻
8. 对无数据支持的股票给强买入

所有 LLM 输出必须落到结构化字段：

- `summary`
- `evidence`
- `confidence`
- `risk`
- `suggestion`

### 10. 当前项目最小可行目标

MVP 成功标准：

每天收盘后自动生成：

1. 半导体细分方向强弱排名
2. 主板可买候选股排名
3. 当前持仓风险评估
4. 明日买入候选
5. 明日卖出候选
6. 每只股票的信号解释

第一阶段不追求自动盈利，先追求：

- 数据正确
- 信号稳定
- 回测可信
- 风控有效
