# V16.0a 行情健康门禁与刷新状态基础能力

## 背景

当前系统能获取行情数据，但缺少健康门禁和只读跟踪闭环。V16.0 建立可观测的实盘行情链路，确保在数据不可靠时 fail closed。

## 需求

### R-001 自选股管理
用户可配置并查看自选股。

### R-002 定时刷新只读行情
- 支持定时刷新
- 刷新失败必须生成去重反馈记录

### R-003 行情健康状态
每条行情展示：来源、行情时间、接收时间、延迟、健康状态。
状态包括：HEALTHY、STALE、UNAVAILABLE、DEMO。

### R-004 数据 fail closed
- allow_demo=False 时不得返回 Demo 数据
- STALE/UNAVAILABLE 时禁止生成正式信号
- Demo 数据必须显著标记，不得冒充实盘

### R-005 确定性信号
- 信号只能由确定性因子、策略和风控规则生成
- LLM 只能解释结果，不能决定买卖
- 本阶段不得创建订单、调用券商或修改账户

### R-006 后台刷新任务状态
任务状态：IDLE、QUEUED、RUNNING、SUCCEEDED、FAILED、CANCELLED

### R-007 API/UI
- API 在后台刷新时仍可响应
- Dashboard 显示最近成功刷新时间、数据状态、确定性信号
- 不得显示下单、确认交易或自动交易按钮
- 用户可见 UI 和 API message 默认中文


## V16.0a 范围说明

本版本（V16.0a）仅交付：
- 行情健康门禁（get_quote_health + 4 状态 + fail-closed）
- 刷新状态基础能力（REFRESH_* 常量 + ServiceManager 方法 + 测试）

以下功能不在 V16.0a 范围，计划 V16.0b：
- API 端点（获取/更新自选股、行情快照、健康状态、启动刷新、刷新状态、信号观测）
- Dashboard 展示（股票代码、最新价、涨跌幅、数据来源等）
- 去重反馈记录
- 自选股存储
- 定时刷新调度
- 确定性信号观测链路集成
