# V15.3 DeepSeek Agent Runtime 功能真实性验证需求

## 1. 背景

PR #3 声称实现了 DeepSeek Agent Runtime 统一框架并完成 BugFixAgent 迁移，但该 PR 长期未合并。V15.3 必须以当前 `main` 为唯一产品基线，验证这些能力是否已经通过其他提交真实进入主线，不能把旧 PR 分支上的代码、描述或测试结果当作当前产品能力。

## 2. 目标

1. 核对当前 `main` 是否存在统一 LLM Runtime 层和 DeepSeek Runtime 封装。
2. 核对结构化输出、只读工具、上下文缓存、会话持久化、用量统计和模型路由委托是否真实实现。
3. 核对 BugFixAgent 是否已经停止直接调用 OpenAI 兼容客户端并迁移到 Runtime。
4. 核对缺少密钥、请求超时和空响应是否默认拒绝继续处理。
5. 核对测试覆盖和交易安全边界。
6. 输出可复现的中文功能说明报告和中文验收报告。

## 3. 验证范围

必须逐项验证以下内容：

1. 是否存在 `src/llm` 或等价 Runtime 层。
2. 是否有 DeepSeek Runtime wrapper。
3. 是否有经过 schema 校验的 JSON 输出。
4. 是否有只读工具注册表和安全护栏。
5. 是否有上下文缓存。
6. 是否有会话持久化。
7. 是否有用量统计。
8. 是否有模型路由委托。
9. BugFixAgent 是否已从直接 OpenAI client 调用迁移到 Runtime。
10. 缺少密钥、请求超时、空内容是否 fail closed。
11. 是否有对应自动化测试。
12. 是否没有新增真实交易能力。
13. 是否不允许 LLM 直接下单。

## 4. 判定规则

- 只检查当前 `main` 的可达代码和测试。未合并 PR、远端旧分支和 PR 描述仅作为对照证据。
- 目录、类名或配置项单独存在，不等于功能完成；必须同时具备实现路径和针对性测试。
- 直接解析 JSON 但不执行 schema 校验，判定为未实现 schema-validated outputs。
- BugFixAgent 只读取 ModelRouter 配置、但仍直接创建 OpenAI client，判定为未迁移。
- 超时没有显式配置和异常收敛，或空响应继续进入下游解析，判定为未满足 fail closed。
- 任一核心项 1 至 11 未实现，整体结论不得为“已实现”。
- 若未实现，验收结论为 `REJECTED`，并建议基于最新 `main` 重新实现；不得直接合并陈旧 PR。

## 5. 非目标

- 不在 V15.3 内实现或移植 DeepSeek Agent Runtime。
- 不修改交易策略、风险、执行、账户、订单、券商或实盘模块。
- 不调用真实 DeepSeek/OpenAI 服务，不消耗真实 API 密钥。
- 不合并 PR #3，不自动合并 `main`。

## 7. 安全约束

- 不触碰 `src/broker/**`、`src/execution/**`、`src/order/**`、`src/account/**`、`src/risk/**`、`miniQMT/**`、实盘交易和真实下单能力。
- 不提交 `.agent/tmp/**` 或 `.agent/reports/**`。
- 所有新增用户可读内容使用中文。
- 证据不足时默认否定能力已实现，不使用 fallback、mock、smoke 或预览产物冒充正式实现。
