# V15.3 DeepSeek Agent Runtime 重实现需求

## 1. 背景

V15.3 真实性验证确认：PR #3 的 DeepSeek Agent Runtime 与 BugFixAgent 迁移没有进入当前 `main`。PR #3 可以作为历史设计与测试参考，但其分支落后当前主线，且仍存在 schema 跳过、缓存未接线、工具失败不够严格、失败路径测试不足等问题，因此本任务从最新 `main` 重新实现。

## 2. 目标

1. 在 `src/llm/` 建立统一 DeepSeek Runtime。
2. 提供异步优先调用、同步兼容入口、超时、有限重试、取消和并发限制。
3. 强制 DeepSeek JSON Output，并对所有成功结果执行 Pydantic 或等价 schema 校验。
4. 支持 Thinking Mode、多轮会话、稳定上下文前缀、用量统计和只读工具调用。
5. 将 BugFixAgent 从直接 OpenAI client 调用迁移到 Runtime。
6. 保持现有研究 Agent 通过 ModelRouter 兼容入口委托 Runtime。
7. 所有缺密钥、缺 SDK、超时、空内容、非法 JSON、未知 schema、工具越权和工具错误均 fail closed。

## 3. 必须功能

| ID | 功能 | 验收标准 |
|---|---|---|
| F-001 | 统一 Runtime | `chat.completions.create` 只允许出现在 Runtime；业务 Agent 不直接创建 client |
| F-002 | 异步执行控制 | 测试覆盖超时、瞬时错误重试、非瞬时错误不重试、取消传播和并发限制 |
| F-003 | Thinking Mode | 请求参数按 profile 设置，原始 reasoning 不进入日志和默认持久化文件 |
| F-004 | 多轮会话 | 会话可追加 system/user/assistant/tool 消息，持久化脱敏并按 TTL 过期 |
| F-005 | 只读工具 | 工具白名单、参数校验、路径约束、输出截断与脱敏；未知或写工具拒绝 |
| F-006 | Context Prefix Cache | 实际调用消息使用缓存生成的稳定前缀；缓存不保存用户 Bug 内容或密钥 |
| F-007 | Schema JSON Output | 设置 `response_format=json_object`；未知 schema、空内容、非法 JSON、字段不匹配均失败 |
| F-008 | BugFixAgent 迁移 | analyze/propose_fix 走 Runtime，仍保留人工审批、受限模块阻断和测试回滚 |
| F-009 | ModelRouter 委托 | 因子发现、研究推荐、信号解释的兼容入口统一委托 Runtime |
| F-010 | 可观测性 | 记录非敏感模型、延迟、token、cache hit/miss、工具轮次和状态 |
| F-011 | 交易安全 | 不新增真实交易能力；LLM 不得直接下单、修改风控或绕过人工确认 |

## 4. PR #3 参考边界

允许参考：

- `src/llm/` 模块拆分。
- fake async client 测试方式。
- Thinking、tool loop、usage 字段映射。
- BugFixAgent public API 保持不变的迁移思路。

不得照搬：

- 未知 schema 静默跳过校验。
- 未知 profile 自动降级继续请求。
- 工具名、参数或执行错误后继续模型循环。
- 不参与实际调用的缓存或会话对象。
- 未覆盖 timeout、retry、cancel、empty content 的“完成”声明。

## 5. 安全约束

- 不修改 `src/risk_engine/**`、`src/execution_engine/**`、`src/broker/**`、`src/order/**`、`src/account/**`、`miniQMT/**` 或真实交易路径。
- ToolRegistry 不注册文件写入、Shell、Git、网络、券商、订单或风控覆盖工具。
- API key 只从环境变量读取，不进入结果、日志、会话或缓存。
- BugFix 提案必须 `requires_approval=true`，分析必须 `needs_human_review=true`。
- `.agent/tmp/**` 和 `.agent/reports/**` 不得提交。
- 不自动合并 `main`。
