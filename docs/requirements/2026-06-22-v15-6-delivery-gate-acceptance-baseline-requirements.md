# V15.6 交付门禁与验收基线修复需求

## 背景

V15.4 报告门禁和 V15.5 中文规范在验收后存在遗留问题：全量测试基线不完整、安全UI未收敛、验收报告需要整改。

## 需求列表

### R-001 修复 V15.4 报告门禁
- Markdown 章节解析必须有实际内容，不能只有标题
- 占位检测只拒绝真正未完成的独立内容
- workflow 必须将 report_gate 结果纳入诊断
- 无论成功失败都上传诊断 artifact

### R-002 完成 V15.5 遗留整改
- AGENTS.md 和 process docs 明确要求中文输出
- handoff 模板要求中文功能说明和验收报告
- 用户可见输出和新增文档默认中文

### R-003 修复安全UI
- 从交易模式下拉框移除 LEVEL_3_AUTO
- 后端拒绝逻辑保留

### R-004 修复测试基线
- 禁止模块导入阶段访问网络
- E2E 测试默认 skip
- 修复时间相关性测试
- 注册 pytest markers

### R-005 历史验收整改
- V15.4/V15.5 复验报告
- 真实测试结果
