# V15.4 PR 报告门禁需求

## 背景

当前 Agent Pipeline 的 PR 验证只检查交易敏感模块和格式，未验证 PR 是否附带必要的功能说明报告和验收报告。开发者可能提交无报告或仅含模板占位的 PR，导致 Reviewer 无法判断变更是否经过充分测试。

## 目标

建立 PR 报告门禁，强制每个非纯文档 PR 在 diff 中附带完整的中文功能说明和验收报告。

## 需求列表

### F-001 纯文档 PR 判定
- 使用 `git diff --name-only <base>...<head>` 获取变更文件列表
- 若全部文件均以 `docs/` 开头且不含 `docs/dev_reports/`、`docs/acceptance/` 以外的 docs 路径，判定为纯文档 PR，跳过报告检查
- 纯文档 PR 允许不附带报告

### F-002 非纯文档 PR 报告要求
- 必须在 diff 中新增或修改 `docs/dev_reports/` 下的中文功能说明报告
- 必须在 diff 中新增或修改 `docs/acceptance/` 下的中文验收报告
- 其他 docs 子目录的变更不满足此要求

### F-003 报告内容要求
报告必须包含以下部分：
- 标题（包含 feature 名称或 PR 编号）
- 变更范围
- 测试命令
- 测试结果
- 安全确认
- 最终结论

### F-004 拒绝规则
触发拒绝的条件：
- 报告文件为空（0 字节）
- 报告内容包含：TODO、TBD、待补充、占位
- 报告内容仅为模板结构无实际内容
- 报告仅为英文占位内容

### F-005 错误处理
- base ref 或 diff 获取失败时，strict 模式必须返回非零退出码
- 报告文件读取失败时，视为缺失
- diff 无变更时，strict 模式可视为纯文档 PR 通过

## 非目标
- 不修改交易敏感模块
- 不修改 Merge Gate 逻辑
- 不自动合并 main
- 不修改 Claude/Codex 执行逻辑
