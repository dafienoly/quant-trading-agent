# V16.0b 只读行情 API/UI、自选股、刷新诊断与信号观测闭环需求

## 延续自 V16.0a

V16.0a 交付了健康门禁和刷新状态基础能力。V16.0b 补齐 API、UI、自选股管理和信号观测链路集成。

## 需求

### R-001 自选股管理
- 读取、更新、校验自选股列表
- 股票代码规范化并去重
- 空列表、非法代码、重复代码有中文错误信息

### R-002 只读行情 API
- 获取自选股、更新自选股
- 获取只读行情快照（含来源、行情时间、接收时间、延迟、健康状态）
- 获取行情健康状态
- 启动一次只读刷新
- 获取刷新任务状态
- 获取确定性信号观测结果

### R-003 后台刷新诊断
- quote_refresh 写入 refresh status
- 成功记录 SUCCEEDED + 时间 + 来源 + symbol 数
- 失败记录 FAILED + 中文错误信息 + evidence
- allow_demo=False 不得用 demo 冒充实盘
- 刷新失败去重反馈（不提交运行产物）

### R-004 Dashboard
- 显示自选股、最新价、涨跌幅、来源、行情时间、延迟、健康状态
- Demo/STALE/UNAVAILABLE 显著中文标记
- 不展示下单等交易按钮
- 用户可见文案中文

### R-005 确定性信号观测
- 仅 HEALTHY 允许生成观测信号
- STALE/UNAVAILABLE/DEMO fail closed
- LLM 仅解释，不决定买卖
- 不创建订单
