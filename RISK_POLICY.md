# 风控政策 (RISK_POLICY.md)

> Risk Agent 拥有系统最高裁决权限，本文档为风控的唯一执行标准。
> 任何 Agent 不得修改、绕过或降级本文档定义的风控规则。

---

## 1. 风控原则

1. **风控优先于收益**：任何信号、策略、交易指令在风险检查未通过时，必须拒绝。
2. **一票否决**：Risk Agent 返回 `risk_pass = false` 时，Execution Agent 必须中断下单流程。
3. **失败即安全**：风控模块自身异常或数据源缺失时，默认禁止交易。
4. **不可配置的硬限制**：下文标注 `[HARD]` 的规则不可通过配置文件更改，必须修改代码并经过完整测试方可调整。

---

## 2. 仓位风险限制

### 2.1 单票仓位 [HARD]

| 持仓类型 | 最大仓位（占账户总资产%） |
|----------|---------------------------|
| 普通持仓 | 15%                       |
| 材料/设备埋伏仓 | 8%                    |
| 光模块/存储弹性仓 | 10%                  |
| 港股通持仓 | 20%                     |

> 计算方式：`仓位 = (持仓数量 x 最新价) / 账户总资产`，汇率换算后计算。

### 2.2 板块集中度 [HARD]

| 板块 | 最大仓位 |
|------|----------|
| 半导体整体 | 60% |
| PCB/CCL | 25% |
| 封测/先进封装 | 20% |
| 设备/材料 | 20% |
| 光模块 | 15% |
| 存储/HBM | 15% |
| 港股半导体 | 20% |

**现金最低比例：20%** [HARD]

## 3. 亏损与回撤控制

### 3.1 单票止损

- 单只股票浮动亏损达到 **-5%**：触发 **减半提醒**，Risk Agent 输出 `action=REDUCE_HALF`。
- 单只股票浮动亏损达到 **-8%**：触发 **强止损提醒**，Risk Agent 输出 `action=STOP_LOSS`，并建议清仓。若 `Trading Mode = LEVEL_2`，允许用户拒绝，但需二次确认。

### 3.2 账户日亏损

- 当日账户累计亏损（含持仓浮动）超过 **2%**：**停止开新仓**，只允许卖出或持有。 [HARD]
- 当日亏损超过 **3%**：**只允许减仓**，不允许任何买入操作。 [HARD]

### 3.3 账户最大回撤

- 从账户历史最高净值回撤超过 **8%**：系统进入 **防守模式**。
  - 防守模式下，新开仓上限降为原标准的 50%，且必须经过人工额外确认。
- 回撤超过 **12%**：**立即停止所有实盘交易**，转入复盘模式，禁止新开仓，仅允许平仓。 [HARD]

---

## 4. 流动性及交易限制

### 4.1 持仓股票

- 日成交额 < 1 亿元的股票，禁止买入。持仓中出现该类股票，仅允许卖出。
- 涨停股票禁止买入，跌停股票禁止卖出。
- 停牌股票禁止任何操作。

### 4.2 黑名单

- 被 Risk Agent 标记的股票进入黑名单，禁止买入。黑名单包括：
  - 被*ST 或 ST
  - 收到监管关注函且未澄清
  - 财务造假嫌疑
  - 近期有大规模减持公告
  - 用户手动加入黑名单

---

## 5. 异常状态监控

Risk Agent 持续监控以下状态，一旦触发立即设置 `risk_pass = false`：

1. 行情数据延迟超过 10 秒（实时行情时间与系统时间比较）。
2. 5 分钟内无任何行情数据更新。
3. 账户资金或持仓与券商返回不一致（偏差超过 1%）。
4. 委托失败次数连续达到 3 次。
5. 网络断开或交易 API 返回异常。
6. 策略信号数量异常（单日 > 50 条或为 0）。
7. 本地时间与交易所时间不同步超过 5 秒。

---

## 6. Kill Switch 触发条件

触发以下任一条件，自动交易系统必须立即停止，且需人工手动恢复：

- 行情数据延迟 > 60 秒
- 账户资产异常（与上次记录偏差 > 5%）
- 连续委托失败 >= 3 次
- 单日亏损 >= 2%
- 个股 1 分钟内涨跌幅异常（+-5% 以上）
- 数据源返回空值或异常值
- 风控模块自身异常
- 用户手动激活 Kill Switch

**Kill Switch 激活后**：

- 撤销所有未成交委托
- 禁止任何新订单生成
- 向用户发送紧急通知
- 所有 Agent 进入只读状态

---

## 7. 交易权限分级风控

| 模式 | 允许操作 | 强制人工确认 |
|------|----------|--------------|
| LEVEL 0 | 数据读取、生成报告 | N/A |
| LEVEL 1 | 生成信号（买入/卖出建议） | N/A |
| LEVEL 2 | 生成委托单 | 是，必须逐笔确认 |
| LEVEL 3 | 自动交易（需满足开启条件） | 否，但 Kill Switch 实时监控 |

> 默认模式：**LEVEL 1**，任何升级必须由用户手动修改配置并重启系统。

---

## 8. 风控检查流程

每个潜在订单必须通过以下顺序检查，全部通过才允许继续：

1. 账户资金检查
2. 单票仓位检查
3. 板块集中度检查
4. 当日亏损检查
5. 持仓亏损检查
6. 流动性检查
7. 黑名单检查
8. 交易时间检查
9. 系统状态检查（数据延迟、网络等）
10. Kill Switch 检查

---

## 9. 日志与审计

- 所有风控决策必须记录日志，包括：时间、订单/信号 ID、检查项、结果、拒绝原因。
- 日志至少保留 5 年，不可删除。
- Risk Agent 自身状态变更（进入防守模式、激活 Kill Switch）必须通过通知渠道（控制台、邮件等）告知用户。
